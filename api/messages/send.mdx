## POST /api/messages/send/:id

Sends a new message to the specified user. Supports both text and image messages. When a message is successfully sent, a `newMessage` Socket.io event is emitted to the receiver if they are online.

### Authentication

This endpoint requires authentication using the `protectRoute` middleware. Include a valid JWT token in the request.

### Path parameters

<ParamField path="id" type="string" required>
  The ID of the user to send the message to (receiver ID)
</ParamField>

### Request body

<ParamField body="text" type="string">
  The text content of the message
</ParamField>

<ParamField body="image" type="string">
  Base64 encoded image data. The image will be uploaded to Cloudinary and the secure URL will be stored
</ParamField>

<Note>
At least one of `text` or `image` must be provided.
</Note>

### Response

<ResponseField name="success" type="boolean">
  Indicates whether the message was sent successfully
</ResponseField>

<ResponseField name="newMessage" type="object">
  The created message object
  
  <Expandable title="Message object">
    <ResponseField name="_id" type="string">
      Unique identifier for the message
    </ResponseField>
    
    <ResponseField name="senderId" type="string">
      ID of the user who sent the message (authenticated user)
    </ResponseField>
    
    <ResponseField name="receiverId" type="string">
      ID of the user receiving the message
    </ResponseField>
    
    <ResponseField name="text" type="string">
      Text content of the message
    </ResponseField>
    
    <ResponseField name="image" type="string">
      Cloudinary URL of the uploaded image (if an image was sent)
    </ResponseField>
    
    <ResponseField name="seen" type="boolean">
      Whether the message has been seen by the receiver (defaults to false)
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp when the message was created
    </ResponseField>
    
    <ResponseField name="updatedAt" type="string">
      ISO 8601 timestamp when the message was last updated
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="message" type="string">
  Error message (only present when success is false)
</ResponseField>

### Socket.io event

When a message is successfully sent, the server emits a `newMessage` event to the receiver's socket (if they are currently online) with the complete message object.

```javascript
socket.on('newMessage', (message) => {
  // Handle incoming message
  console.log('New message received:', message);
});
```

### Example request

```bash
curl -X POST https://api.quickchat.com/api/messages/send/507f1f77bcf86cd799439011 \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Hello! How are you?"
  }'
```

### Example request with image

```bash
curl -X POST https://api.quickchat.com/api/messages/send/507f1f77bcf86cd799439011 \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Check out this image!",
```
